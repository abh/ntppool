Come mi unisco a pool.ntp.org?

Prima di tutto: Grazie per il tuo interesse. L'uso del pool è cresciuto enormemente e l'unico modo per mantenere facile l'accesso a tutti al pool server è di aumentare anche il numero di server partecipanti.

Il tuo computer deve avere un indirizzo IP statico e una connessione internet permanente. È molto importante che il tuo indirizzo IP non cambi o almeno che lo faccia molto di rado (diciamo una volta all'anno o meno). La larghezza di banda richiesta è relativamente poca. Ogni client invierà solo un paio di pacchetti UDP ogni pochi (fino a 20) minuti.

Attualmente la maggior parte dei server riceve circa 5-15 pacchetti NTP per secondo con picchi un paio di volte al giorno di 60-120 pacchetti per secondo. Ciò equivale approssimativamente a 10-15Kbit/s con picchi di 50-120Kbit/s. Il progetto acquisisce costantemente più timeserver, quindi il carico non dovrebbe incrementare drammaticamente per ogni server. In parole semplici, ti serviranno almeno 384-512Kbit di banda (in ingresso e in uscita).

I server che aderiscono non dovrebbero avere pool.ntp.org come loro server, ma dovrebbero configurare alcuni buoni server manualmente (questi server andrebbero scelti dal pool. Il punto è che dovrebbero essere scelti staticamente invece che assegnati casualmente ad ogni riavvio del server. Questo aiuterà a fornire un servizio di qualità accettabile). Nota che non è necessario che il tuo server sia un server stratum 1 o 2 - dato che questo progetto distribuisce gran parte del carico, non c'è motivo per cui un stratum 3 o persino stratum 4 non dovrebbe unirsi.

Abbiamo una pagina con le raccomandazioni sulla configurazione per i server che si uniscono al pool.

Infine, devo enfatizzare che unirsi al pool è un impegno a lungo termine. Saremo felici di rimuoverti dal pool di nuovo se le tue circostanze cambiassero, ma dato il funzionamento dei client ntp saranno necessarie settimane, mesi o persino ANNI prima che il traffico sparisca completamente.

Se va tutto bene, effettua il login alla pagina gestione server e richiedi che il tuo server venga aggiunto. Se dovessi avere problemi con il sistema, inviami una email a ask@develooper.com.

Ti chiederei anche di sottoscriverti ai feed weblog e magari anche alla mailing list pool.

Sarebbe carino (ma non necessario) se tu potessi redirigere le richieste web sulla porta 80 del tuo time server alla pagina web ufficiale del progetto su http://www.pool.ntp.org. Se esegui Apache, puoi farlo usando:

[% PROCESS tpl/join/virtual_host_example.html %]

Di nuovo, questo solo se esegui un webserver. La pagina web ufficiale del progetto sarà sempre fornita con il 'www' all'inizio - ma a volte la gente è pigra e scrive semplicemente pool.ntp.org e restano sorpresi di ricevere una pagina web casuale.

Una volta aggiunto al pool, il server viene costantemente monitorato per disponibilità e precisione. Puoi osservare le performance del tuo server usando l'interfaccia web o la pagina di gestione.
