Wie benutze ich den pool.ntp.org?

Wenn du nur deine Computeruhr mit dem Netzwerk synchronisieren willst, ist die Konfigurationdatei (für ntpd der ntp.org Distribution auf jedem unterstützten Betriebssystem - Linux, *BSD, Windows und sogar ein paar noch exotischere Systeme) wirklich einfach: 

driftfile /var/lib/ntp/ntp.drift

server 0.pool.ntp.org
server 1.pool.ntp.org
server 2.pool.ntp.org
server 3.pool.ntp.org

Die Namen 0, 1 und 2.pool.ntp.org zeigen auf ein zufälliges Set von stündlich wechselnden Servern. Stell sicher, dass deine Computeruhr empfindlich eingestellt ist (innerhalb weniger Minuten der "richtigen" Zeit). Du kannst ntpdate pool.ntp.org oder einfach den Befehl date benutzen und deine Armbanduhr stellen. Starte ntpd und nach einer Weile (das kann bis zu einer halben Stunde dauern!), ntpq -pn. Das gibt sieht dann etwa so aus: 

avbidder:~$ ntpq -p
     remote           refid      st t when poll reach   delay   offset  jitter
==============================================================================
+81.6.42.224     193.5.216.14     2 u   68 1024  377  158.995   51.220  50.287
*217.162.232.173 130.149.17.8     2 u  191 1024  176   79.245    3.589  27.454
-129.132.57.95   131.188.3.222    3 u  766 1024  377   22.302   -2.928   0.508

Die IP-Adressen werden anders sein, weil du zufälligen Zeitservern zugeteilt wirst. Das Wichtige ist, dass eine der Linien mit einem Sternchen (*) startet. Das bedeutet, dein Computer bekommt die Zeit aus dem Internet - Du mußt dich nie wieder darum kümmern!

Weil pool.ntp.org dir Zeitserver aus der ganzen Welt zuteilt, wird die Zeitqualität nicht ideal sein. Du bekommst bessere Ergebnisse, wenn du Kontinentalzonen benutzt (z.B. Europa, Nordamerika, Ozeanien oder Asien.pool.ntp.org), und sogar noch bessere wenn du Länderzonen benutzt (wie ch.pool.ntp.org in der Schweiz). Für alle diese Zonen kannst du wieder die Prefixe 0, 1 oder 2 benutzen. Beachte aber, dass die Länderzone für dein Land vielleicht nicht existiert oder dass es vielleicht nur einen oder zwei Zeitserver gibt. Wenn du Zeitserver kennst, die sehr nahe sind (gemeßen an der Netzwerkdistanz, mit traceroute oder ping), wird die Zeit vermutlich noch besser sein.

Wenn du eine aktuelle Windows-Version benutzt, kannst du den eingebauten ntp Client benutzen. Gib einfach folgendes in der Befehlszeile ein: 

net time /setsntp:pool.ntp.org

In manchen Windows-Versionen kannst du auch mehr als einen Server angeben: 

net time /setsntp:"0.pool.ntp.org 1.pool.ntp.org 2.pool.ntp.org"

Dies funktioniert sicher auf Windows 2000/XP/2003. Das Gleiche kann man auch als Administrator erreihen, indem man die Uhr in der Taskbar mit Rechts anklickt, Uhrzeit&Datum einstellen wählt und den Zeitserver unter "Internetzeit" eingibt.

Meinberg hat auch eine Portierung des ntp daemon fü Windows gemacht.

Wenn dein Windows-System teil einer Domäne ist, kannst du vielleicht nicht eigenständig die Zeit deines Computers ändern. Siehe Wie der Windows Time Service funktioniert.

Zusätzliche Bemerkungen

Wenn du eine statische IP-Adresse und eine vernünftige Internetverbindung hast (Bandbreite ist nicht so wichtig, aber sie sollte stabil sein und zu überladen), bitte überlege, deinen Server dem Serverpool beizusteuern. Es kostet dich nicht mehr als ein paar hundert Byte pro Sekunde und du kannst dem Projekt helfen, fortzubestehen. Lies die Beitrittsseite für mehr Informationen.

Wenn dein Internet-Provider einen Zeitserver hat, oder du einen guten Zeitserver in deiner Nähe kennst, solltest du diesen benutzen und nicht die Liste - du bekommst so vermutlich eine bessere Zeit und benutzt weniger Netzwerkressourcen. Wenn du nur einen Zeitserver in deiner Nähe kennst kannst du natürlich diesen und zwei aus pool.ntp.org benutzen.

Es kann selten passieren, dass du dem gleichen Zeitserver zwei mal zugeordnet wirst. Starte einfach den ntp Server neu, das löst das Problem normalerweise. Wenn du eine Länderzone benutzt, bitte beachte, dass es sein kann, dass in diesem Land nur ein Server dem Projekt bekannt ist. Benutze lieber Kontinentalzonen in diesem Fall. Du kannst die Zonen durchsuchen um zu sehen, wie viele Server wir pro Zone haben.

Sei freundlich. Viele Server sind durch Freiwillige bereitgestellt und meistens sind diese Server in Wirklichkeit Datei-, Mail- oder Webserver die einfach nur auch ntp betreiben. Bitte benutze nicht mehr als drei Zeitserver in deiner Konfiguration und spiel kein schmutziges Spiel mit burst oder minpoll. Alles, was du damit vielleicht erreichst ist, dass das Projekt früher oder später aufhört.

Stell sicher, dass die Zeitzone deines Computers richtig eingestellt ist. ntpd selbst macht nichts mit der Zeitzone, es benutzt intern nur UTC.

Falls du ein Netzwerk mit pool.ntp.org synchronisierst, bitte stell einen Computer als Server ab und synchronisier die anderen mit diesem. (Du musst ein bißchen lesen, aber ist nicht sehr schwer. Und es gibt immer die comp.protocols.time.ntp Newsgroup)

An dieser Stelle möchte ich allen danken, die ihre Zeit und ihre Zeitserver dem Projekt widmen.
