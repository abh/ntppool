Wie kann ich pool.ntp.org beitreten?

Zu erst: Danke für dein Interesse. Die Nutzung des Pools ist enorm gewachsen und die einzige Möglichkeit, es für alle einfach zu halten einen Pool-Server zu betreiben ist es, auch die Zahl der teilnehmenden Pool-Server zu erhöhen.

Dein Computer muss eine statische IP-Adresse besitzen und eine permanente Internetverbindung. Es ist sehr wichtig dass die IP-Adresse sich nicht ändert oder nur sehr sehr selten (einmal im Jahr seltener). Die benötigte Bandbreite ist realtiv gering. Jeder Client wird nur einige wenige UDP-Pakete senden alle paar bis all 20 Minuten.

Derzeit erhalten die meisten Server zwischen fünf und 15 NTP-Pakete pro Sekunde mit mehrmaligen Spitzen täglich von 60-120 Paketen pro Sekunde. Dies ist etwa so viel wie 10-15kbit/s mit Spitzen von 50-120kbit/s Das Projekt akquiriert ständig neue Zeitserver, so dass der Load für jeden Server nicht dramatisch größer werden sollte. Kurz gesagt, du brauchst mindestens 384-512Kbit Bandbreite (ein- und ausgehend).

Hier einige Server-Trafficload-Grafiken: [% i = 0; FOR server = combust.servers_with_urls; %] [% i=i+1; i %] [% END %]

Die beitretenden Server sollten nicht pool.ntp.org als upstream server haben, sondern mit einem guten Server manuell konfiguriert werden. (Diese Server können aus dem Pool gewählt werden. Der Punkt ist, dass diese statistisch gewählt werden anstelle einer zufälliger Zuordnung, wenn der Server neu startet. Dies hilft, die Qualität des Dienstes sicherzustellen.) Beachte dass es ist nötig ist, dass dein Server ein stratum 1 oder 2 Server ist. Da das Projekt den Load größtenteils verteilt, gibt es keinen Grund, warum ein stratum 3 oder stratum 4 Server nicht auch mitmachen könnte.

Wir haben eine Seite mit Konfigurationsempfehlungen für beitretende Server.

Schlussendlich muss ich betonen, dass der Beitrtitt zum Pool eine langfristige Verpflichtung darstellt. Wir nehmen dich gerne wieder aus dem Pool wenn die Umstände sich für dich ändern, aber aufgrund der Arbeitsweise der NTP-Clients, dauert es Wochen, Monae oder sogar JAHRE bevor der Traffic komplett verschwindet.

Wenn dies alles OK ist, logge dich auf der Server Management-Seite ein und beantrage deinen Sevrer hinzuzufügen. Wenn du Probleme mit dem System hast, schreibe mir eine E-Mail auf ask@develooper.com.

Ich bitte auch darum, den Weblog Feed zu abonnieren und vielleicht auch die Pool Mailingliste.

Es wäre schön (aber nicht notwendig), wenn du Webanfragen auf Port 80 deines Zeitservers auf die offizielle Webseite des Projekts auf http://www.pool.ntp.org umleitest. Wenn du Apache benutzt, kannst du das so machen:

[% PROCESS tpl/join/virtual_host_beispiel.html %]

Nochmals, dies gilt nur, wenn du sowieso einen Webserver betreibst. Die offizielle Projekt-Webseite wird immer mit "www" am Anfang, nur manchmal sind die Leute faul und tippen nur pool.ntp.org ein, sind dann erstaunt, dass sie auf einer zufälligen Seite landen.

Einmal zum Pool hinzugefügt wird der Server konstant auf Präzision und Verfügbarkeit überwacht. Du kannst die Performance deines Servers über das Web-Frontend oder die Management-Seite ansehen.
