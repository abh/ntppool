Configuratieaanbevelingen voor servers die deelnemen in de pool

De Support sectie op support.ntp.org heeft heel veel nuttige informatie.

Wanneer het de bedoeling is om alleen de pool te gebruiken, zie dan de pool gebruik pagina.

De comp.protocols.time.ntp nieuwsgroep is de beste plek om hulp te krijgen voor de ntpd software.

Hieronder staan een aantal zaken die van belang zijn wanneer je met jouw server wilt deelnemen in de NTP Pool.

Management queries

Make the default configuration be to not allow "management queries". For ntpd this will be adding the "noquery" option to the default "restrict" lines, for example:

[% INCLUDE "join/configuration/restrict-default.html" %]

To allow commands like "ntpq -c pe" to work from localhost you can add:

[% INCLUDE "join/configuration/restrict-localhost.html" %]

Configureer ongeveer 5 servers

Om goed te werken moet ntpd op zijn minst met drie servers communiceren ("Een man met een klok weet wat de tijd is. Een man met twee klokken is daar nooit zeker van").

Voor servers in de pool adviseren we om niet minder dan 4 en niet meer dan 7 servers te configureren.

Gebruik geen *.pool.ntp.org servers

Ironisch genoeg is het om er voor de zorgen dat de pool service optimaal werkt niet de bedoeling om de *.pool.ntp.prg aliases in de configuratie op te nemen wanneer die server aan de pool deel gaat nemen.

Voor de robuustheid van de pool is het gezonder wanneer alle pool beheerders "met de hand" goede lokale (qua netwerk gezien) tijdservers uitkiezen. De NTP.org wiki heeft een lijst van publieke servers.

Gebruik de standaard ntpd

We zijn allen voor software diversiteit, maar een significant percentage van de "het werkt niet" vragen die binnenkomen zijn voor andere software dan ntpd.

Je kunt de pool gebruiken met elk programma dat NTP spreekt, maar wanneer je aan de pool gaat deelnemen bevelen we aan om ntpd te gebruiken.

Gebruik niet de LOCAL klok driver

Servers in de NTP pool moeten niet de LOCAL klok driver hebben geconfigureerd.
