Configuració recomanada per a servidors que s'uneixin al pool

La secció de "suport" a support.ntp.org conté molta informació útil.

Si només voleu fer servir el pool, mireu la pàgina sobre l'ús.

El grup de discussió comp.protocols.time.ntp és el millor lloc per obtenir ajuda amb el programari ntpd.

Més a baix trobareu algunes coses d'interès particular, si voleu unir el vostre servidor al pool.

Management queries

Make the default configuration be to not allow "management queries". For ntpd this will be adding the "noquery" option to the default "restrict" lines, for example:

[% INCLUDE "join/configuration/restrict-default.html" %]

To allow commands like "ntpq -c pe" to work from localhost you can add:

[% INCLUDE "join/configuration/restrict-localhost.html" %]

Configureu aproximadament 5 servidors

Perquè funcioni correctament l'ntpd ha de comunicar-se amb 3 servidors com a mínim ("Un home amb un rellotge sap quina hora és. Un home amb dos rellotges mai no n'està segur.").

Per a servidors del pool recomanem configurar com a mínim 4 i com a màxim 7 servidors.

No feu servir servidors de *.pool.ntp.org

Semblarà irònic, però per assegurar-nos de que el pool funcioni de la millor manera possible no feu servir *.pool.ntp.org a la vostra configuració, si us proposeu d'afegir el vostre servidor al pool.

Per tenir un pool robust és millor si tots els operadors trien bons servidors de temps locals (respecte a la xarxa) de manera manual. El wiki de NTP.org conté una llista de servidors públics.

Feu servir l'ntpd estàndard

Tots estem a favor de diversitat de programari, però un percentatge significant de les preguntes a l'estil "no funciona" que ens arriben es refereixen a programari que no és l'ntpd.

Podeu fer servir el pool amb qualsevol programa que sap parlar NTP, però si voleu unir el vostre servidor al pool us recomanem que feu servir ntpd.

No feu servir el controlador de rellotge "LOCAL"

Servidors del pool no haurien de tenir configurat el controlador de rellotge "LOCAL".

Pareu atenció a les consultes DNS PTR

Alguns administradors dels servidors han manifestat que alguns usuaris del ntp (o el tallafocs que facin servir) fa una consulta "reverse DNS" (PTR) a l'adreça IP del servidor NTP. Teniu en compte que això provocarà un augment de les consultes. Es recomana augmentar el "temps de vida" de les consultes si això suposa un problema, ja que farà baixar el nombre de consultes a un nivell més baix.
