El grafico de puntuación

Un par de veces cada hora el servidor principal comprueba la hora de tu servidor y lo compara con la hora local. La puntuación baja si no se puede contactar con el servidor o si la diferencia de tiempo es mayor a 100ms (como se miden los servidores de monitorización). La puntuación baja más si la diferencia es mayor.

Cuando la puntuación baja, el color de fondo en la parte de arriba del gráfico reflejará proporcionalmente la bajada de puntos. La escala de color va de azul (muy pequeño) a amarillo, naranja y rojo (el servidor lleva varios segundos apagado o es inalcanzable). La forma de calcular la media del gráfico, no se puede saber porque ha pasado el cambio de color. El color sólo importa como una herramienta para comparar servidores. Para detalles exactos de como lo hace el sistema de monitorización puedes hacer click en el enlace CSV.

El gráfico de diferencia

El sistema de monitorización funciona estimando al cliente SNTP (RFC 2030), así que es muy susceptible a la latencia de la red aleatoria entre el servidor y el sistema de monitorización, más que un cliente ntp regular puede ser. En otras palabras, no te preocupes por las diferencias grandes ocasionales y por favor no uses en absoluto la diferencia como medida de optimización del reloj.
