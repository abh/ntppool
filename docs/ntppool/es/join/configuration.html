Recomendaciones de configuración para los sevidores que se unen a la lista de servidores

La sección de soporte en support.ntp.org tiene mucha información útil.

Si simplemente quieres utilizar la lista de sevidores, echa un vistazo a la página de uso de la lista de servidores.

El grupo de noticias comp.protocols.time.ntp es el mejor lugar para obtener ayuda del programa ntpd (servicio de protocolo de hora de Internet).

Debajo te ofrecemos algunas cuestiones de particular importancia si vas a unirte a la lista de servidores NTP con tu servidor.

Management queries

Make the default configuration be to not allow "management queries". For ntpd this will be adding the "noquery" option to the default "restrict" lines, for example:

[% INCLUDE "join/configuration/restrict-default.html" %]

To allow commands like "ntpq -c pe" to work from localhost you can add:

[% INCLUDE "join/configuration/restrict-localhost.html" %]

Configura unos 5 servidores

Para que el servicio ntpd funcione debidamente necesitas conectar con 3 servidores ("Un hombre con un reloj sabe la hora que es. Un hombre con 2 relojes nunca está seguro").

Para los servidores añadidos a la lista de servidores recomendamos configurar no menos de 4 servidores y no más de 7 servidores.

No uses los servidores *.pool.ntp.org

Ironicamente, para asegurarse de que el servicio del grupo es el mejor posible, no deberías usar los alias *.pool.ntp.org en tu configuración cuando vas a añadir tu servidor a la lista de servidores.

Para que la lista de servidores sea robusta, es mejor que todos los servidores de la lista "elijan a dedo" buenos servidores horarios locales (a esto se le llama red inteligente). La wiki de NTP.org tiene una lista de servidores públicos.

Uso del servicio de protocolo hora de red estándar (ntpd)

Todos apostamos por la diversidad de programas, pero un porcentaje significativo de problemas del tipo "no funciona" que aparecen en ocasiones se producen con software distinto a ntpd.

Puedes utilizar la lista de servidores con cualquier programa que hable NTP, pero si vas a participar en el proyecto, te recomendamos que uses ntpd.

No utilices el controlador de reloj LOCAL

Los servidores del grupo NTP no deben tener configurado el controlador de reloj LOCAL.
