Recomendaciones de configuración para los sevidores que se unen a la lista de servidores

La sección de soporte en support.ntp.org tiene mucha información util.

Si simplemente quieres usar la lista de sevidores, echa un vistazo a la página uso de la lista de servidores.

El grupo de noticias comp.protocols.time.ntp es el mejor lugar para obtener ayuda del programa ntpd (servicio de protocolo de hora de internet).

Debajo hay algunas cosas de particular importancia si vas a unirte a la lista de servidores NTP con tu servidor.

Configuración con 5 servidores

Para que el servicio ntpd funcione debidamente necesitas conectar con 3 servidores ("Un hombre con un reloj sabe la hora que es. Un hombre con 2 relojes nunca está seguro").

Para los servidores en la lista de servidores recomendamos configurar entre 4 y 7 servidores horarios.

No uses los servidores *.pool.ntp.org

Ironicamente, para asegurarse de que el servicio del grupo es el mejor posible, no deberías usar los alias *.pool.ntp.org en tu configuración cuando vas a añadir tu servidor a la lista de servidores.

Para que la lista de servidores sea consistente es mejor que todos los servidores de la lista recojan a dedo buenos servidores horarios locales (a esto se le llama red inteligente). La wiki de NTP.org tiene una lista de servidores publicos.

Uso del servicio de protocolo hora de red estándar (ntpd)

Todos apostamos por la diversidad de programas, pero un porcentaje significativo de temas del tipo "no funciona" que vienen son de programas que no son el ntpd.

Puedes usar la lista de servidores con cualquier programa que tranaje con NTP, pero si vas a participar en el proyecto te recomendamos que uses ntpd.

No uses el driver de reloj local

Los servidores del grupo NTP no deberían tener el driver del reloj local configurado.
