¿Como me uno al grupo pool.ntp.org?

Lo primero, gracias por tu interés. El uso de la lista de servidores ha crecido en grandes proporciones y la única forma para todos de mantener de una manera sencilla el administrar la lista de servidores para incluso que crezca el numero de servidores participantes.

Tu ordenador/servidor debe tener una dirección IP estática y una conexión permanente a internet. Es muy importante que la dirección IP no cambie o que, si lo hace, sea con una frecuencia extremadamente mínima (dígase una vez al año o menos). El ancho de banda requerido es más bien bajo. Cada cliente sólo enviará un par de paquetes UDP cada poco, como mucho unos 20 minutos.

Por lo general muchos servidores recogen entre 5 y 15 paquetes NTP por segundo con picos un par de veces al día de 60 a 120 paquetes por segundo. Esto es más o menos equivalente a unos 10-15 Kbit/segundo (1-2KB/s) con picos de unos 50-120Kbit/segundo (6-15KB/s). El proyecto constantemente adquiere más servidores de hora, así que la carga de red no debería incrementar mucho por cada servidor. Es decir, probablemente necesites como mínimo un ancho de banda de unos 384-512Kbit (48-64KB/s) entre bajada y subida.

Los servidores participantes no deberían tener configurado a pool.ntp.org como servidor, pero se deberían configurar algunos buenos servidores manualmente (estos servidores podrían ser elegidos desde la lista de servidores. La idea es que se eligen estaticamente en lugar de asignarse automáticamente para cada reinicio del servidor. Esto ayudará a proveer de una calidad aceptable del servicio.) Piensa que no se requiere que tu servidor sea de 1er o 2o nivel - como este proyecto trata de distribuir la carga, no hay porqué de ser de nivel 3 o incluso un servidor de nivel 4 el que deba participar.

Tenemos una pagina con recomendaciones para la configuración para servidores que se unen a la lista de seridores.

Finalmente, dedo poner énfasis en que la participación en el grupo es un compromiso de larga duración. También nos congratulará sacarte de la lista si tus circunstancias cambian, pero debido a como operan los clientes ntp pueden pasar semanas, meses o incluso años antes de que el tráfico desaparezca completamente.

Si estás deacuerdo con todo, inicia sesión en la página de administración de servidor y pide que se añada tu servidor a la lista. Si tienes algún problema con el sistema, mándame un correo a ask@develooper.com.

También te pediría que te suscribas al weblog y si puede ser a la lista de correo del proyecto.

Sería recomendable (pero no necesario) si pudieras redireccionar las peticiones web al puerto 80 de tu servidor harario a la pagina web oficial del proyecto en http://www.pool.ntp.org. Si ejecutas Apache, puedes hacerlo con:

[% PROCESS tpl/join/virtual_host_example.html %]

Reitero lo dicho, realízalo sólo si ejecutas un servidor web de algun modo. La página web oficial del proyecto siempre es con las tres 'w' del principio (www) - pero a veces la gente pasa y solo escribe pool.ntp.org y se sorprende al ver que les sale una página web aleatoria.

Una vez añadido a la lista el servidor se monitoreará constantemente para comprobar su disponibilidad y precisión. Puedes ver el rendimiento de tu servidor echandole un vistazo a la portada de la web o la página de administración.
