Wie trete ich dem NTP Pool bei?

Zu allererst: Herzlichen Dank für das Interesse. Die Benutzung des Pools hat stark zugenommen. Um diese Benutzung auch in Zukunft möglichst einfach gestalten zu können, muss die Zahl der teilnehmenden Server ebenfalls steigen.

Ihr Computer muss über eine statische IP-Adresse verfügen und eine permanente Verbindung mit dem Internet aufweisen. Es ist zwingend erforderlich, dass sich die Adresse Ihres Servers nur äußerst selten (weniger als ein mal im Jahr) ändert. Die benötigte Bandbreite ist relative gering gehalten, da jeder Klient nur alle paar Minuten ein paar UDP-Packete anfordert.

Aktuell werden die meisten Server mit etwa 5-15 NTP Packeten pro Sekunden, mit Spitzenwerten von von ca. 60-120 Packeten pro Sekunde belastet. Grob gerechnet entspricht das einer Datenrate von 10-15Kbit/sec bzw. 50-120Kbit/sec. Dem Pool werden ständig neue Zeitserver hinzugefügt, so dass für die Datenrate auch in Zukunft kein starker Anstieg zu erwarten ist. Kurz gesagt: Sie benötigen eine Bandbreite von 384-512Kbit/s (ein- und ausgehen).

Teilnehmede Server sollten pool.ntp.org nicht als Zeitquelle angegeben haben, sollten jedoch einen guten Zeitserver benutzen. Der Grund ist, dass die Zeitserver bei jedem Start statistisch unabhängig voneinander ausgewählt werden. Dadurch soll eine akzeptable Qualität der Zeitsynchronisation gewärleistet werden. Ihr Server muss dazu kein Stratum 1 oder 2 sein, da der NTP Pool die Last auf eben diese schließlich auf viele Server verteilen will. Für Stratum 3 oder 4 gibt es keinen Grund, nicht dem Pool beizutreten.

Es wird darauf hingewiesen, dass die Teilnahme am NTP Pool eine langfristige Entscheidung sein sollte. Gerne können wir Ihren Server auch aus dem Pool entfernen, doch auf Grund der technischen Umsetzung des NTP Pools, kann es Wochen, Monate bis sogar Jahre dauern, bis kein Klient mehr Anfragen an Sie richtet.

Für die Teilnahme begeben Sie sich bitte auf die Seiten des Server Management and beantragen Sie Ihre Teilnahme. Sollten dennoch Schwierigkeiten entstehen, wenden Sie sich bitte direkt an mich: ask@develooper.com.

Außerdem bitte ich Sie den the Weblog und eventuell die Pool Mailing Liste zu abonnieren.

Es wäre nett (aber nicht erfoderlich) von Ihnen, wenn Sie Anfragen auf Port 80 zur offiziellen Webseite des Projektes http://www.pool.ntp.org weiterleiten könnten. Wenn Sie Apache einsetzten konfigurieren Sie das durch:

[% PROCESS tpl/join/virtual_host_example.html %]

Die offizielle Webseite des Projekts wird immer unter www.pool.ntp.org erreichbar sein, jedoch zwingt sich dem faulen Menschen auf, das www schlicht zu ignorieren. Um Überraschungen zu vermeiden, wäre eine Weiterleitung daher hilfreich.

Einmal Hinzugefügt, wird ihr Server kontinuierlich überwacht und die Genauigkeit ihres Zeitservers untersucht. Die Ergebnisse können Sie unter auf der Management Seite oder dem Web Frontend erfahren.
