Рекомендации по конфигурированию серверов, входящих в пул

Очень много полезной информации содержит раздел поддержки сайта support.ntp.org.

Если вы собираетесь просто воспользоваться услугами пула для синхронизации времени (а не участвовать в нем), вам сюда.

В случае каких-либо проблем с настройкой и использованием ntpd, обращайтесь в конференцию comp.protocols.time.ntp.

Ниже приведены несколько важных советов, которые нужно учитывать при настройке входящих в пул серверов.

Management queries

Make the default configuration be to not allow "management queries". For ntpd this will be adding the "noquery" option to the default "restrict" lines, for example:

[% INCLUDE "join/configuration/restrict-default.html" %]

To allow commands like "ntpq -c pe" to work from localhost you can add:

[% INCLUDE "join/configuration/restrict-localhost.html" %]

Используйте порядка 5 серверов

Для устойчивой работы ntpd необходимо как минимум 3 сервера (есть такая пословица: «Тот, у кого есть часы, всегда знает точное время. Тот, у кого двое часов, никогда не знает точного времени»).

Для серверов, входящих в пул, рекомендуется использовать не менее 4, но не более 7 серверов.

Не используйте сервера *.pool.ntp.org

В интересах обеспечения стабильной и эффективной работы пула, не используйте псевдонимы *.pool.ntp.org в качестве источников точного времени для серверов, входящих в пул.

Для устойчивой работы пула выбирайте точные, близкие к вам (по сетевой дистанции) сервера вручную из списков на wiki NTP.org.

Используйте стандартный ntpd

Мы не возражаем против разнообразия программного обеспечения, однако большинство жалоб «оно не работает» приходится на различные альтернативы ntpd.

Вы можете пользоваться услугами пула при помощи любого NTP-клиента, однако если вы собраетесь в нем участвовать, мы рекомендуем выбрать ntpd.

Не используйте локальный источник времени (LOCAL)

Сервера, участвующие в пуле NTP, не должны использовать LOCAL как источник времени.
