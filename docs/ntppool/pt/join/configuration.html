Recomendações de configuração para juntar servidores ao pool

A seção de suporte em support.ntp.org possui muitas informações úteis.

Se você simplesmente deseja usar o pool, veja a página uso do pool.

O grupo de notícias comp.protocols.time.ntp é o melhor lugar para obter ajuda com o software ntpd.

Abaixo estão algumas coisas de importância particular, se você vai juntar o NTP Pool com seu servidor.

Management queries

Make the default configuration be to not allow "management queries". For ntpd this will be adding the "noquery" option to the default "restrict" lines, for example:

[% INCLUDE "join/configuration/restrict-default.html" %]

To allow commands like "ntpq -c pe" to work from localhost you can add:

[% INCLUDE "join/configuration/restrict-localhost.html" %]

Configure cerca de 5 servidores

Para funcionar adequadamente, o ntpd precisa conversar com pelo menos 3 servidores ("Um homem com um relógio sabe que horas são, um homem com dois relógios nunca tem certeza").

Para servidores no pool, nós recomendamos configurar não menos de 4 e não mais de 7 servidores.

Não use servidores *.pool.ntp.org

Ironicamente, para assegurar de que o serviço do pool seja o melhor possível, você não deveria usar os endereços *.pool.ntp.org na sua configuração quando você estiver adicionando seu servidor ao pool.

Para a robustez do pool é mais saudável que todos os administradores de pool escolham "a dedo" bons servidores de tempo locais (com boa resposta de rede). O wiki do NTP.org tem uma lista de servidores públicos.

Use o ntpd padrão

Somos todos pela diversidade de softwares, mas uma porcentagem significativa das perguntas "não está funcionando" que chegam são relacionados a softwares sem ser ntpd.

Você pode usar o pool com qualquer programa que fale NTP, mas se você for se juntar ao pool, nós recomendamos que você use o ntpd.

Não use um driver de relógio LOCAL

Os servidores no NTP Pool não deveriam ter o driver de relógio LOCAL configurado.
