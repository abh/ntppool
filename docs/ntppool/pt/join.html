Como posso fazer para me juntar ao pool.ntp.org?

Primeiramente, obrigado pelo seu interesse. O uso do pool tem aumentado muito e a única forma que nós podemos fazer para facilitar para todo muito executar um servidor do pool é também aumentar o número de servidores participando.

Seu computador deve ter um endereço IP estático e uma conexão internet permanente. É muito importante que seu endereço IP não seja alterado ou que isso apenas ocorra raramente (digamos, uma vez no ano ou menos até). A largura de banda necessária é relativamente baixa. Cada cliente vai enviar um conjunto de pacotes UDP a cada 20 minutos.

Atualmente, a maioria dos servidores recebem cerca de 5-15 pacotes NTP por segundo com alguns picos por dia de 60-120 pacotes por segundo. Isso é basicamente equivalente a 10-15Kbits/segundo com picos de 50-120Kbits/segundo. O projeto constantemente angaria mais servidores de tempo, de forma que a carga não deve aumentar dramaticamente para cada servidor. Em resumo, você provavelmente precisará de uma largura de banda de 384-512Kbit (de entrada e saída).

Os servidores que estão se juntando não devem ter pool.ntp.org como seus servidores upstream, e sim deveriam configurar alguns servidores bons manualmente (aqueles servidores podem ser escolhidos do pool. A questão é que eles são escolhidos estaticamente em vez de atribuído aleatoriamente a cada inicialização do servidor. Isso vai ajudar a fornecer uma qualidade aceitável do serviço.) Note que não é necessário que seu servidor seja um servidor stratum 1 ou 2 - como este projeto é sobre em sua maioria distribuição de carga, não há motivos para que um servidor stratum 3 ou até mesmo um stratum 4 não possa se juntar.

Nós temos uma página com recomendações de configuração para servidores que estão se juntando ao pool.

Finalmente, nós enfatizamos que juntar-se ao pool é um compromisso a longo prazo. Nós podemos prontamente tirar você do pool novamente se suas circunstâncias mudarem, mas por causa da forma que os clientes ntp operam podem levar semanas, meses ou até ANOS antes que o tráfego se vá por completo.

Se estiver tudo ok, autentique-se na página de gerenciamento de servidor e solicite que seu servidor seja adicionado. Se você tiver qualquer problema com o sistema, envie um e-mail para mim em ask@develooper.com.

Eu gostaria de solicitar que você também se inscreva ao feed do weblog e, se possível, à lista de discussão do pool.

Seria legal (mas não é necessário) se você pudesse redirecionar requisições web na porta 80 do seu servidor de tempo para a página web oficial do projeto em http://www.pool.ntp.org. Se você usa Apache, você poderia fazê-lo com:

[% PROCESS tpl/join/virtual_host_example.html %]

Novamente, isso apenas se você usa algum servidor web. A página web oficial do projeto sempre será fornecida com o "www" no começo - mas algumas vezes as pessoas são preguiçosas e apenas digitam pool.ntp.org e são surpreendidas ao alcançar uma página web aleatória.

Uma vez adicionado ao pool, o servidor é constantemente monitorado por disponibilidade e precisão. Você pode monitorar a performance do seu servidor usando o frontend web ou a página de gerenciamento.
