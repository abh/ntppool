Cum mă alătur la pool.ntp.org?

Mai întâi: Mulțumim pentru interesul dumneavoastră. Utilizarea pool-ului a crescut enorm, iar singura cale prin care putem păstra ușurința pentru toată lumea de a rula un server de pool este să creștem și numărul de servere participante.

Computerul dumneavoastră trebuie să aibă o adresă IP statică și o conexiune permanentă la Internet. Este foarte important ca adresa IP să nu se schimbe sau să o facă extrem de rar (să zicem o dată pe an sau mai puțin). Lățimea de bandă necesară este relativ mică. Fiecare client va trimite doar câteva pachete UDP la fiecare câteva minute până la 20 de minute.

În prezent, majoritatea serverelor primesc aproximativ 5-15 pachete NTP pe secundă cu vârfuri de 60-120 pachete pe secundă de câteva ori pe zi. Aceasta este aproximativ echivalentă cu 10-15Kbit/sec cu vârfuri de 50-120Kbit/sec. Proiectul dobândește constant mai multe servere de timp, așa că încărcătura nu ar trebui să crească dramatic pentru fiecare server. În termeni simpli, probabil aveți nevoie de cel puțin 384-512Kbit lățime de bandă (intrare și ieșire).

Serverele care se alătură nu ar trebui să aibă pool.ntp.org ca serverele lor de referință, ci ar trebui să configureze manual niște servere bune. Acele servere pot fi alese din pool. Ideea este că acestea sunt alese static în loc să fie atribuite aleatoriu din pool la fiecare repornire a serverului. Aceasta va ajuta la furnizarea unei calități acceptabile a serviciului. Rețineți că nu este necesar ca serverul dumneavoastră să fie un server stratum 1 sau 2 - deoarece acest proiect se referă în principal la distribuția încărcăturii, nu există niciun motiv pentru care un server stratum 3 sau chiar stratum 4 să nu se alăture.

Avem o pagină cu recomandări de configurare pentru serverele care se alătură pool-ului.

În final, trebuie să subliniez că alăturarea la pool este un angajament pe termen lung. Suntem bucuroși să vă scoatem din pool din nou dacă circumstanțele se schimbă, dar din cauza modului în care operează clienții NTP va dura săptămâni, luni sau chiar ANI înainte ca traficul să dispară complet.

Dacă totul este în regulă, conectați-vă la pagina de gestionare a serverului și solicitați ca serverul dumneavoastră să fie adăugat. Dacă aveți probleme cu sistemul, trimiteți-mi un email la ask@develooper.com.

Vă rugăm să vă alăturați și la forumul comunității

Ar fi frumos (dar nu este necesar) dacă ați putea redirecționa cererile web de pe portul 80 al serverului dumneavoastră de timp către pagina web oficială a proiectului la https://www.ntppool.org/. Dacă rulați Apache, ați putea face aceasta cu:

[% PROCESS tpl/join/virtual_host_example.html %]

Din nou, aceasta doar dacă rulați oricum un server web. Pagina web oficială a proiectului va fi întotdeauna dată cu 'www' la început - dar uneori oamenii tastează pool.ntp.org și sunt apoi surprinși să primească o pagină web aleatorie.

Odată adăugat la pool, serverul este monitorizat constant pentru disponibilitate și precizie. Puteți urmări performanța serverului dumneavoastră folosind interfața web sau pagina de gestionare.
