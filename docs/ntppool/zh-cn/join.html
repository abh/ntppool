如何加入 pool.ntp.org?

首先: 感谢您对此感兴趣。使用pool服务的客户群在巨大的增长,我们只有一个办法来 保证所有人都能轻松容易的使用pool服务，那就是不断增加参与的服务器的数量。

您的服务器 必须拥有一个静态IP地址 和一个持久稳定的网络连接。 这一点非常重要，对于您的IP地址来说是不能改变的或者非常少频率的变更 (一年一次或几年一次)。 项目需求的带宽非常低，每个客户端每20分钟只是发送一对UDP数据包。

当前多数服务器获得每秒 5-15 个NTP包和每天间隔数次的每秒60-120个左右的数据包。 这大致相当于 10-15Kbit/秒 和 50-120Kbit/秒。这个项目拥有稳定数量的时间服务器， 所以每个服务器的负荷不会很高。用专业术语来说，您需要至少384-512Kbit的带宽 (上传和下行).

这台加入的服务器不应该将 pool.ntp.org 作为其上游服务器, 但是应该手动配置一些 good servers (这些服务器也可以从pool选择，重要的一点是它们因该是静态随机的组合在一起而不是 每次服务器重启时随机获得,这将有助于提供一个优质的服务。) 注意，这并不是必须要求您的服务器是1层或2层的设计，这个项目主要是的负载的分配。 没有什么原因会拒绝3层或4层的服务器的加入。

关于配置 configuration recommendations for servers joining the pool 的页面.

最后，我必须要强调的是：加入pool 应该是一个 长期的承诺。 如果您的情况发生变化我们也欢迎您退出pool。但是由于不同ntp客户端 的设置 这个过程会持续数周, 几个月甚至一年直至流量变为零.

如果已经做完了这些, 请登录 server management 这个页面并提交加入您服务器的请求. 您有任何关于系统方面的问题, 都可以发邮件到 ask@develooper.com.

我们建议您订阅weblog feed 或者订阅 pool mailing list.
