كيف يمكنني الاشتراك في pool.ntp.org؟

أولا: شكرا لك على الاهتمام. استخدام هذه المجموعة في ازدياد مستمر والطريقة الوحيدة لجعلها بسيطة للجميع في تشغيلها هي زيادة أعداد الخوادم المشاركة.

يجب أن يكون لجهازك عنوان IP ثابت إضافة إلى خدمة انترنت دائمة. من المهم ألا يتغير عنوان IP أبدا أو يتغير مرة في السنة على الأكثر. سعة الخط المطلوبة قليلة جدا. طلب المزامنة الواحد لا يأخذ أكثر من عدة بايتات مرة بين بعض دقائق و ساعة واحدة.

حاليا أغلب الخوادم تحصل على ما بين خمسة و خمس عشرة رزمة NTP في الثانية مع إمكانية وصولها ما بين 60 و 120 رزمة في الثانية في بعض الأوقات. وهذا تقريبا يساوي استخدام ما بين 10 و 15 كيلوبت في الثانية مع إمكانية وصولها ما بين 50 و 120 كيلوبت في الثانية. يكسب هذا المشروع خوادم جديدة بشكل ثابت مع الوقت، وهذا يعني أن الخادم الواحد so the load should not increase dramatically for each server. بشكل مبسط، تحتاج لما بين 384 و 512 كيلوبت في الثانية من سعة خط الانترنت لتقديم الخدمة بشكل طبيعي.

فيما يلي بعض الخوادم مع رسوماتها البيانية التي توضح حركة وأحمال البيانات: [% i = 0; FOR server = combust.servers_with_urls; %] [% i=i+1; i %] [% END %]

The servers joining should not have pool.ntp.org as their upstream server(s), but should configure some good servers manually (those servers may be chosen from the pool. The point is that they are chosen statically instead of assigned randomly at each server restart. This will help providing an acceptable quality of service.) Note that it is not required that your server is a stratum 1 or 2 server - as this project is about load distribution mostly, there is no reason why a stratum 3 or even stratum 4 server shouldn't join.

We have a page with configuration recommendations for servers joining the pool.

Finally, I must emphasize that joining the pool is a long term commitment. We are happy to take you out of the pool again if your circumstances change, but because of how the ntp clients operate it will take weeks, months or even YEARS before the traffic completely goes away.

If this is all ok, login to the server management page and request your server to be added. If you have any trouble with the system, email me at ask@develooper.com.

I'd ask you to also subscribe to the weblog feed and maybe to the pool mailing list.

It would be nice (but it's not necessary) if you could redirect web requests on port 80 of your time server to the official web page of the project at https://www.ntppool.org. If you run Apache, you could do this with:

[% PROCESS tpl/join/virtual_host_example.html %]

Again, this is only if you run a webserver anyway. The official project web page will always be given with the 'www' at the beginning - but sometimes people type pool.ntp.org and are then surprised to get a random web page.

Once added to the pool the server is constantly monitored for availability and precision. You can watch your server's performance using the web frontend or the management page.
