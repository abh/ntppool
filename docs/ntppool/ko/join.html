[% page.title = 'NTP 풀에 합류하십시오!' %]

어떻게 하면 pool.ntp.org에 합류합니까?

우선 관심을 보여 주셔서 감사합니다. 풀의 사용은 엄청나게 늘어났고 모든 사람들에게 지속적으로 서비스를 제공하기 위한 유일한 방법은 참여하는 서버들의 숫자를 같이 늘리는 것입니다.

귀하의 컴퓨터는 반드시 고정 IP 주소를 갖고 있어야 하고 인터넷에 언제나 연결되어 있어야 합니다. 귀하의 IP 주소는 변하지 않거나 아주 가끔 (일년에 한 번 이하) 변해야 합니다. 요구 대역폭은 상대적으로 적습니다. 각 클라이언트들은 아주 가끔이나 20분에 한번씩 몇 개의 UDP 패킷을 보낼 것입니다.

현재 대부분의 서버들은 초당 5~15개의 NTP 패킷을 받고 있고 초당 60~120개의 패킷을 받는 일이 하루에 몇 번 정도 있습니다. 이것은 대략 초당 10~15Kbit 이고 최대 초당 50~120Kbit 입니다. 이 프로젝트는 지속적으로 서버를 늘려나가므로 각 서버당 부하량이 갑자기 늘어나는 일은 없을 것입니다. 쉽게 말해 귀하는 최소 384~512Kbit 의 (양방향) 대역폭이 필요합니다.

몇 개 서버의 대역폭/부하 그래프는 다음과 같습니다. [% i = 0; FOR server = combust.servers_with_urls; %] [% i=i+1; i %] [% END %]

합류하는 서버에 pool.ntp.org를 상위 서버로 설정해서는 안됩니다. 대신 좋은 서버들 중에서 골라서 설정하시면 좋습니다. (혹시 이 서버들이 풀에 존재할 수도 있는데 매번 서버가 재실행될 때마다 무작위로 상위 서버를 지정하는 것 보다는 이렇게 고정적으로 설정하는 것이 서비스 품질을 유지하는 데에 도움이 됩니다.) 귀하의 서버가 1이나 2계층의 서버라면 이럴 필요가 없습니다. 이 프로젝트의 주 목적은 부하 분산이므로 3계층이나 4계층의 서버들도 합류를 막을 이유가 없기 때문입니다.

풀에 합류하는 서버의 설정 권장 사항 페이지를 참고하십시오.

마지막으로 이 풀에 합류하는 것은 장기적인 협약이라는 것을 강조하고 싶습니다. 상황이 변해서 귀하가 풀에서 서버를 제거하는 것은 어쩔 수 없지만 NTP 클라이언트들의 동작 방식으로 인해 트래픽이 완전히 사라지기까지는 보통 몇주나 몇달, 심한 경우에는 몇년이 걸릴수도 있습니다.

모두 받아들일 준비가 되었다면 <서버 관리 페이지에 가서 로그인을 하고 서버를 추가하십시오. 시스템에 문제가 있가면 ask@develooper.com 주소로 저에게 이메일을 보내십시오.

웹로그 피드와 시간 지키미 공지사항 메일링 리스트에도 가입할 것을 권장합니다.

(별로 필요한 사항은 아니지만) 귀하의 시간 서버의 80번 포트로 오는 웹 요청을 본 프로젝트 공식 홈페이지 http://www.pool.ntp.org 로 리디렉션을 해 주신다면 고맙겠습니다. 아파치를 사용하고 계시면 이렇게 설정하십시오:

[% PROCESS tpl/join/virtual_host_example.html %]

이것은 귀하가 웹 서버를 운영하고 있는 경우에만 해당됩니다. 프로젝트의 공식 웹 페이지는 앞에 'www' 가 언제나 붙습니다만 사람들 중에는 게을러서 그냥 pool.ntp.org 만 치고는 아무 웹 페이지가 나와서 깜짝 놀라는 사람들도 있기 때문입니다.

일단 풀에 서버가 추가되면 가용성과 정확성을 위해 지속적으로 모니터링됩니다. 서버의 상태는 웹 화면이나 관리 페이지에 나와 있습니다.
