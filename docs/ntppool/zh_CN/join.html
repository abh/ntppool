我如何加入pool.ntp.org?

首先，感谢您对此感兴趣。地址池的使用量增长十分巨大，唯一一个可以让每个人都能容易的运行地址池服务器的方法，就是增加参与的服务器数量。

你的电脑必须有一个固定的IP地址和一个长期稳定的互联网连接。保证你的IP地址不会发生变化是非常重要的，或者仅变化极少几次（一年一次甚至更低）。所需要的带宽比较少，每个客户端在每20分钟里，只会发送几个UDP数据包。

通常大多数服务器平均每秒会收到大约5-15个NTP包，高峰期大约每秒60-120个。这大致相当于10-15Kbit/s，高峰期50-120Kbit/s。这个项目将逐渐得到更多的服务器，因此每个服务器的负载应该不会出现明显的提高。简单的说，你大约需要至少384-512Kb的带宽（流入和流出）。

这里是几台服务器的流量/负载监控图： [% i = 0; FOR server = combust.servers_with_urls; %] [% i=i+1; i %] [% END %]

加入的服务器不应该使用pool.ntp.org作为其上游服务器（们），而应该手动配置一些优质的服务器作为他们的上游服务器。（这些服务器可能是从地址池当中选出来的。关键点在于它们是被固定选中的，而非在每个服务器重新启动的时候随机选择的。这有助于提供一个可以接受的服务质量。） 注意，我们并不要求你的服务器是第1-2层的服务器 —— 因为这个项目是负载均衡相关的，3-4层的服务器是没有不应该加入的理由的。

我们设立了一个加入地址池的服务器推荐配置的页面。

最后，我需要强调的是加入地址池是一个长期的承诺。如果你的情况发生了变化，我们很愿意再将你移除出地址池，但是由于ntp客户端的操作方式，想让流量完全消失可能会消耗数周数月甚至数年的时间。.

如果这些都是可以的话，请到服务器管理页面登录，并请求加入你的服务器。 如果你对这个系统有任何的问题，请通过邮件联系我：ask@develooper.com。

我也会让您订阅 博客feed 或许以及 地址池邮件列表.

一旦加入了地址池，服务器就会被持续监控它的可用性以及精确度。你可以通过网页前台或者管理页面查看你的服务器的表现。
