我如何才能加入 pool.ntp.org?

首先感谢您对我们有兴趣。NTP群组的使用量增加得很快， 为了使更多的人能使用这个服务，最好的方式是加入我们，使您的服务器为其他人提供服务。

您的电脑 必须拥有一个静态IP地址 与永久性的网络连接。请记住： 您应该拥有一个静态的IP地址 或者只是极少改变您的IP地址（一年改变一次或更少）。 项目所需的带宽不会很多。 每个客户端只会每20分钟发送几个UDP数据包

目前大多数服务器每秒接收到大约5-15个NTP数据包， 一天中的几次繁忙时段将会是每秒60-120个数据包。 这大致相当于10-15Kbit/sec，峰值时50-120Kbit/sec。 这个群组有越来越多的服务器加入，所以每一台服务器的负载不会极具增加 在正常条件下，您可能需要最少384-512Kbit的上下行带宽。

这是一张某台服务器的流量/负载图表： [% i = 0; FOR server = combust.servers_with_urls; %] [% i=i+1; i %] [% END %]

加入计划的服务器不能把 pool.ntp.org 用作上游服务器，需要手动设置一些 准确服务器 (这些服务器 可以 从池中选取。 （这些服务器应该被手动选择，而不是每次重启后随机分配，这将有助于提供高质量的服务。) 请注意，我们要求您的服务器是第1层(stratum)或第2层(stratum)服务器 - 由于此项目主要涉及负载分配，因此第3层(stratum)甚至第4层(stratum)服务器不应该加入。

我们有一个页面有关 如何配置NTP服务器.

最后，我必须强调：加入NTP群组是一个 长期承诺。 我们很乐意当您的情况改变时将你移出群组，但是由于ntp 客户端的原理， 访问流量完全消失可能要耗费数周、数月，甚至数年！.

如果以上条件您完全符合，请访问 服务器管理 页面，并且申请将您的服务器加入群组。 如果您对这个系统有任何问题，请给我发送电子邮件： ask@develooper.com.

我想邀请您订阅 网页博客 以及可能的话还有 NTP群组邮件列表.
