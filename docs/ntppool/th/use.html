ฉันจะใช้ pool.ntp.org ได้อย่างไร?

หากคุณต้องการเพียงแค่ซิงโครไนซ์นาฬิกาของคอมพิวเตอร์กับเครือข่าย ไฟล์กำหนดค่า (สำหรับโปรแกรม ntpd จากชุดซอฟต์แวร์ของ ntp.org บนระบบปฏิบัติการที่รองรับ เช่น Linux, *BSD, Windows และระบบปฏิบัติการเฉพาะอื่น ๆ) ก็สามารถทำได้อย่างง่ายดาย

[% INCLUDE "ntppool/use/sample-config.html" %]

ชื่อ 0, 1, 2 และ 3.pool.ntp.org ชี้ไปยังชุดเซิร์ฟเวอร์แบบสุ่มที่จะเปลี่ยนทุกชั่วโมง ตรวจสอบให้แน่ใจว่านาฬิกาของคอมพิวเตอร์ของคุณตั้งค่าใกล้เคียงกับเวลาจริง (ภายในไม่กี่นาทีของเวลา 'จริง') - คุณสามารถใช้ ntpdate pool.ntp.org หรือใช้คำสั่ง date และตั้งค่าตามนาฬิกาข้อมือของคุณ เริ่ม ntpd และหลังจากผ่านไประยะหนึ่ง (อาจนานถึงครึ่งชั่วโมง!) ntpq -pn ควรแสดงผลดังนี้

[% INCLUDE "ntppool/use/sample-ntpq.html" %]

IP แอดเดรสจะแตกต่างกันเพราะคุณได้รับเซิร์ฟเวอร์เวลาแบบสุ่ม สิ่งสำคัญคือหนึ่งในบรรทัดจะเริ่มต้นด้วยเครื่องหมายดอกจัน (*) ซึ่งหมายความว่าคอมพิวเตอร์ของคุณได้รับเวลาจากอินเทอร์เน็ตแล้ว และคุณไม่ต้องกังวลกับเรื่องนี้อีกต่อไป

การค้นหา pool.ntp.org (หรือ 0.pool.ntp.org, 1.pool.ntp.org ฯลฯ) มักจะส่งคืน IP แอดเดรสสำหรับเซิร์ฟเวอร์ในหรือใกล้ประเทศของคุณ สำหรับผู้ใช้ส่วนใหญ่ สิ่งนี้จะให้ผลลัพธ์ที่ดีที่สุด

คุณยังสามารถใช้โซนทวีปได้ (ตัวอย่างเช่น europe, north-america, oceania หรือ asia.pool.ntp.org) และโซนประเทศ (เช่น ch.pool.ntp.org ในสวิตเซอร์แลนด์) - สำหรับโซนเหล่านี้ทั้งหมด คุณสามารถใช้คำนำหน้า 0, 1, 2 หรือ 3 ได้อีก เช่น 0.ch.pool.ntp.org อย่างไรก็ตาม โปรดทราบว่าโซนประเทศอาจไม่มีสำหรับประเทศของคุณ หรืออาจมีเซิร์ฟเวอร์เวลาเพียงหนึ่งหรือสองเครื่อง

โปรดทราบด้วยว่าปัจจุบันระบบจะให้ IPv6 แอดเดรสสำหรับโซนนอกเหนือจาก IPv4 แอดเดรสก็ต่อเมื่อชื่อโซนมีคำนำหน้าเลข 2 เช่น 2.pool.ntp.org (หากมีเซิร์ฟเวอร์ NTP IPv6 ในโซนนั้น) ชื่อโซนที่ไม่มีคำนำหน้าเลข หรือมีคำนำหน้า 0, 1 หรือ 3 ปัจจุบันให้เฉพาะ IPv4 แอดเดรสเท่านั้น

หากคุณใช้ Windows เวอร์ชันใหม่ คุณสามารถใช้ไคลเอนต์ NTP ที่มีอยู่ในระบบ

คลิกขวาที่วันที่และเวลาในพื้นที่ Taskbar และเลือก Adjust Date/time หรือเปิดการตั้งค่าด้วย Win+I และคลิก Time & Language -> Date & Time เลื่อนลงและคลิก Date, time, regional formatting เลื่อนลงและคลิก Additional date, time & regional settings คลิก Date and Time และ Internet Time คลิกปุ่ม Change settings ป้อน pool.ntp.org และคลิก Update now

ทดสอบโดยคลิกขวาที่วันที่เวลาในพื้นที่ระบบและเลือก Adjust Date/time หรือเปิดการตั้งค่าด้วย Win+I และคลิก Time & Language -> Date & Time คลิกปุ่ม "Sync now"

หากระบบ Windows ของคุณเป็นส่วนหนึ่งของโดเมน คุณอาจไม่สามารถอัปเดตเวลาของคอมพิวเตอร์ได้อย่างอิสระ สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการตั้งค่าเวลาบน Windows โปรดดู How Windows Time Service Works

หมายเหตุเพิ่มเติม

พิจารณาว่า NTP Pool เหมาะสมกับการใช้งานของคุณหรือไม่ หากธุรกิจ องค์กร หรือชีวิตมนุษย์ขึ้นอยู่กับการมีเวลาที่ถูกต้องหรืออาจได้รับอันตรายจากเวลาที่ผิดพลาด คุณไม่ควร "เพียงแค่ดึงเวลาจากอินเทอร์เน็ต" NTP Pool โดยทั่วไปมีคุณภาพสูงมาก แต่เป็นบริการที่ดำเนินการโดยอาสาสมัครในเวลาว่าง กรุณาพูดคุยกับผู้จำหน่ายอุปกรณ์และบริการของคุณเกี่ยวกับการตั้งค่าบริการท้องถิ่นและเชื่อถือได้สำหรับคุณ โปรดดูข้อกำหนดการให้บริการของเราด้วย เราแนะนำเซิร์ฟเวอร์เวลาจาก Meinberg แต่คุณยังสามารถหาเซิร์ฟเวอร์เวลาจาก End Run, Spectracom และอื่น ๆ อีกมากมาย

หากคุณมี IP แอดเดรสคงที่และการเชื่อมต่ออินเทอร์เน็ตที่เหมาะสม (แบนด์วิดท์ไม่สำคัญมากนัก แต่ควรเสถียรและไม่มีโหลดสูงเกินไป) กรุณาพิจารณาบริจาคเซิร์ฟเวอร์ของคุณให้กับพูลเซิร์ฟเวอร์ มันไม่ได้ทำให้คุณเสียค่าใช้จ่ายมากกว่าสองสามร้อยไบต์ต่อวินาทีของทราฟฟิก แต่คุณช่วยให้โครงการนี้อยู่รอด กรุณาอ่านหน้าการเข้าร่วมสำหรับข้อมูลเพิ่มเติม

หากผู้ให้บริการอินเทอร์เน็ตของคุณมีเซิร์ฟเวอร์เวลา หรือหากคุณรู้จักเซิร์ฟเวอร์เวลาที่ดีใกล้คุณ คุณควรใช้เซิร์ฟเวอร์นั้นและไม่ใช้รายการนี้ - คุณอาจได้เวลาที่ดีกว่าและใช้ทรัพยากรเครือข่ายน้อยลง หากคุณรู้จักเซิร์ฟเวอร์เวลาใกล้คุณเพียงหนึ่งเครื่อง คุณสามารถใช้เซิร์ฟเวอร์นั้นและอีกสองเครื่องจาก pool.ntp.org ได้

บางครั้งอาจเกิดขึ้นที่คุณได้รับเซิร์ฟเวอร์เวลาเดิมสองครั้ง - การรีสตาร์ทเซิร์ฟเวอร์ NTP มักจะแก้ปัญหานี้ได้ หากคุณใช้โซนประเทศ โปรดทราบว่าอาจเป็นเพราะมีเซิร์ฟเวอร์เพียงเครื่องเดียวที่รู้จักในโครงการ - ในกรณีนั้นควรใช้โซนทวีปแทน คุณสามารถเรียกดูโซนเพื่อดูจำนวนเซิร์ฟเวอร์ที่เรามีในแต่ละโซน

เป็นมิตร เซิร์ฟเวอร์จำนวนมากให้บริการโดยอาสาสมัคร และเซิร์ฟเวอร์เวลาเกือบทั้งหมดเป็นไฟล์เซิร์ฟเวอร์หรือเมลเซิร์ฟเวอร์หรือเว็บเซิร์ฟเวอร์ที่บังเอิญใช้ NTP ด้วย ดังนั้นอย่าใช้เซิร์ฟเวอร์เวลาเกินสี่เครื่องในการกำหนดค่าของคุณ และอย่าใช้เทคนิค burst หรือ minpoll - สิ่งที่คุณจะได้คือโหลดเพิ่มบนเซิร์ฟเวอร์เวลาอาสาสมัครเท่านั้น

ตรวจสอบให้แน่ใจว่าการกำหนดค่าโซนเวลาของคอมพิวเตอร์ของคุณถูกต้อง ntpd เองไม่ทำอะไรเกี่ยวกับโซนเวลา มันใช้ UTC ภายในเท่านั้น

หากคุณกำลังซิงโครไนซ์เครือข่ายกับ pool.ntp.org กรุณาตั้งค่าคอมพิวเตอร์เครื่องหนึ่งของคุณเป็นเซิร์ฟเวอร์เวลาและซิงโครไนซ์คอมพิวเตอร์เครื่องอื่นกับเครื่องนั้น (คุณต้องอ่านบ้าง - มันไม่ยากหรอก และยังมี กลุ่มข่าว comp.protocols.time.ntp เสมอ)
